## Setting Up React From Zero
vite
### Step 1 — Install Node.js

React needs Node.js. Check if you have it:

```bash
node -v
npm -v
```

If not installed, download from **nodejs.org** (get the LTS version).

---

### Step 2 — Create a React Project

The modern way is **Vite** (not Create React App — CRA is outdated and slow):

```bash
npm create vite@latest my-app -- --template react
cd my-app
npm install
npm run dev
```

Your app runs at `http://localhost:5173`

## What Each File/Folder Actually Does

### `index.html` — The Entry Point of the Browser
```html
<div id="root"></div>
<script type="module" src="/src/main.jsx"></script>
```
React mounts your entire app into that single `<div id="root">`. You almost never touch this file.

---

### `src/main.jsx` — Where React Starts
```jsx
import { StrictMode } from 'react'
import { createRoot } from 'react-dom/client'
import './index.css'
import App from './App.jsx'

createRoot(document.getElementById('root')).render(
  <StrictMode>
    <App />
  </StrictMode>,
)
```
This grabs the `root` div and injects your `App` component into it. **You rarely edit this.**

---

### `src/App.jsx` — Your Root Component
This is where your actual app starts. Everything you build goes inside or is imported here.

---

### `src/index.css` — Global Styles
CSS that applies to the whole app.

---

### `src/assets/` — Static Files
Images, SVGs, fonts, etc.

---

### `node_modules/` — Installed Packages
All your dependencies live here. **Never touch this manually.** It's auto-generated by `npm install`.

---

### `package.json` — Project Config
Lists your dependencies and scripts:
```json
"scripts": {
  "dev": "vite",        // npm run dev → starts local server
  "build": "vite build" // npm run build → production build
}
```

### `vite.config.js` — Build Tool Config
Vite settings. You'll rarely need to touch this early on.

## Real-World Folder Structure (How You Should Grow It)

The default structure is fine for small apps. As your app grows, organize `src/` like this:

```
src/
├── assets/              # images, fonts, icons
├── components/          # reusable UI components
│   ├── Navbar.jsx
│   ├── Button.jsx
│   └── Card.jsx
├── pages/               # full page components
│   ├── Home.jsx
│   ├── About.jsx
│   └── Login.jsx
├── hooks/               # custom hooks
│   └── useFetch.js
├── context/             # global state (Context API)
│   └── AuthContext.jsx
├── services/            # API calls
│   └── api.js
├── utils/               # helper functions
│   └── formatDate.js
├── App.jsx
├── index.css
└── main.jsx
```

### Why This Structure?

| Folder | Purpose |
|---|---|
| `components/` | Small, reusable pieces — Button, Navbar, Modal |
| `pages/` | Full pages composed of components — Home, Dashboard |Pages = combination of components

| `hooks/` | Your custom React hooks |
| `context/` | Shared global state across components |
| `services/` | All API/fetch logic in one place |
| `utils/` | Pure JS helper functions |

---

## The Rule to Follow

> **Components go in `components/`, pages go in `pages/`, logic goes in `hooks/` or `services/`.**

Don't dump everything in `src/` root. You'll regret it when the app grows.

# ⚙️ How App Connects Everything

### main.jsx

```jsx
import React from "react";
import ReactDOM from "react-dom/client";
import App from "./App";

ReactDOM.createRoot(document.getElementById("root")).render(
  <App />
);
```

---

### App.jsx

```jsx
import Home from "./pages/Home";

function App() {
  return <Home />;
}

export default App;
```

---

### Home.jsx

```jsx
import Navbar from "../components/Navbar";
import HeroBanner from "../components/HeroBanner";

function Home() {
  return (
    <>
      <Navbar />
      <HeroBanner />
    </>
  );
}

export default Home;
```
